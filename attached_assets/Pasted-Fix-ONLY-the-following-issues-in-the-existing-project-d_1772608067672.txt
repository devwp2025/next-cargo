ISSUE 1 — Admin KYC Approve/Reject does not actually update the user (API mismatch)
In server/routes.ts, the existing admin KYC endpoint is:
  PATCH /api/admin/users/:id/kyc
and it validates body as:
  { kycStatus: "unverified" | "pending" | "approved" | "rejected" }

But the UI is currently triggering a POST request like:
  POST /api/admin/users/:id/kyc
and/or sending a different payload (e.g. { action: "approve" }).
As a result, approve/reject appears to do nothing.

Fix (choose ONE approach, minimal change):
A) Update the frontend admin users page to call the existing endpoint correctly:
   - Use PATCH /api/admin/users/:id/kyc
   - Send JSON body: { kycStatus: "approved" } or { kycStatus: "rejected" }
   - Ensure Content-Type: application/json
   - After success, update the row in state or refetch the list

OR

B) Extend the backend to also accept POST with { action: "approve" | "reject" }:
   - Map approve -> kycStatus="approved", reject -> kycStatus="rejected"
   - Keep the existing PATCH behavior unchanged
   - Return the updated user object (safe user) in response

Requirement:
- Approve/Reject must immediately reflect in the admin UI without manual refresh.
